Wrapping Up

This guide showed you how to deploy a simple AWS Lambda based application with Octopus. The deployment was implemented across multiple individual projects that:

  • Deployed a shared API Gateway instance.
  • Deployed a Lambda exposing a REST API via API Gateway.
  • Deployed a static website hosted in S3 and exposed via API Gateway.

We also created the associated Octopus resources required to support these deployments, including:

  • An AWS account.
  • The Development and Production environments.
  • A Maven feed to consume the application artifacts.
  • A library variable set containing shared variables.

All of these resources were created via Terraform, forming the basis of a repeatable and customizable workflow that can be modified to suit your own internal requirements.

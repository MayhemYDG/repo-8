Deploying the Frontend Web App

The frontend web app provides the interface end users consume to view the library of books and blog posts maintained by the Octopub sample application.

Unlike the products service, which is deployed as a Lambda, the frontend web app is a static web site with no server side processing. This allows us to host the frontend web app in an S3 bucket. API Gateway is then configured to direct requests to S3, providing end users with a single hostname and shared URL structure for accessing both the static frontend website and Lambda based backend APIs.

To create the frontend web application deployment project in Octopus, apply the Terraform configuration in the terraform/frontend/octopus directory using the following commands: 

cd terraform/frontend/octopus
terraform init
terraform apply

You are prompted for the URL of your Octopus instance, the Octopus API key, the ID of the space to apply the configuration to, the AWS access key, and the AWS secret key. Terraform then displays a list of changes and prompts you to apply them. Once the configuration is applied a new project called Frontend WebApp is created. Breaking down the HCL

The process of deploying a Lambda and exposing it via API Gateway is:

  • Query the CloudFormation stack that deployed the API Gateway instance for outputs like the API Gateway ID and the ID of the shared resources attached to the gateway.
  • Create an S3 bucket to host the static web application.
  • Upload the web application to the S3 bucket.
  • Proxy web requests from the API Gateway to the static web endpoint exposed by S3.
  • Update the API Gateway stage.
  • Log the URL used to access the frontend web application.

The project makes use of structured configuration variables, which is a feature of Octopus that allows specially named variables to be injected into files such as JSON, YAML, XML, and Java properties files.

This is the default JSON used to configured the frontend web app: 

{
  "basename": "",
  "branch": "main",
  "title": "Octopub",
  "productEndpoint": "/api/products",
  "productHealthEndpoint": "/health/products",
  "auditEndpoint": "/api/audits",
  "auditHealthEndpoint": "/health/audits",
  "overrideTheme": "",
  "frontendVersion": "v0.1.0"
}

Normally, relative paths like /api/products would work, but because API Gateway stages append the name of the stage to the path (for example Development in the URL https://apigatewayid.execute-api.ap-southeast-2.amazonaws.com/Development). This means the default relative paths do not work, because the relative path /api/products would result in requests to paths like https://apigatewayid.execute-api.ap-southeast-2.amazonaws.com/api/products, which doesn't work.

This means we must replace the JSON properties called productHealthEndpoint and productEndpoint with a new value reflecting the URL exposed by API Gateway.

Here we create a variable that will overwrite the property called productHealthEndpoint in a JSON configuration file with the URL of the API Gateway stage exposing the backend API health check: 

resource "octopusdeploy_variable" "frontend_webapp_producthealthendpoint" {
  owner_id     = "${octopusdeploy_project.project_frontend_webapp.id}"
  value        = "https://#{Octopus.Action[Get Stack Outputs].Output.RestApi}.execute-api.ap-southeast-2.amazonaws.com/#{Octopus.Environment.Name}/health/products"
  name         = "productHealthEndpoint"
  type         = "String"
  description  = ""
  is_sensitive = false

  scope {
    actions      = []
    channels     = []
    environments = []
    machines     = []
    roles        = null
    tenant_tags  = null
  }
  depends_on = []
}

The final step builds up the URL that the frontend web application is accessed via and displays it in the deployment logs: 

    action {
      action_type                        = "Octopus.AwsRunScript"
      name                               = "Get Stage Outputs"
      condition                          = "Success"
      run_on_server                      = true
      is_disabled                        = false
      can_be_used_for_project_versioning = false
      is_required                        = false
      worker_pool_id                     = "${data.octopusdeploy_worker_pools.workerpool_hosted_ubuntu.worker_pools[0].id}"
      properties                         = {
        "Octopus.Action.Script.Syntax" = "Bash"
        "Octopus.Action.Aws.Region" = "#{AWS.Region}"
        "OctopusUseBundledTooling" = "False"
        "Octopus.Action.Script.ScriptBody" = <<-EOF
        echo "Downloading Docker images"

        echo "##octopus[stdout-verbose]"

        docker pull amazon/aws-cli 2>&1

        # Alias the docker run commands
        shopt -s expand_aliases
        alias aws="docker run --rm -i -v $(pwd):/build -e AWS_DEFAULT_REGION=$AWS_DEFAULT_REGION -e AWS_ACCESS_KEY_ID=$AWS_ACCESS_KEY_ID -e AWS_SECRET_ACCESS_KEY=$AWS_SECRET_ACCESS_KEY amazon/aws-cli"

        echo "##octopus[stdout-default]"

        STAGE_URL=$(aws cloudformation \
            describe-stacks \
            --stack-name "OctopubApiGatewayStage-#{Octopus.Environment.Name | Replace " .*" ""}" \
            --query "Stacks[0].Outputs[?OutputKey=='StageURL'].OutputValue" \
            --output text)

        set_octopusvariable "StageURL" $${STAGE_URL}
        echo "Stage URL: $STAGE_URL"

        DNS_NAME=$(aws cloudformation \
            describe-stacks \
            --stack-name "OctopubApiGatewayStage-#{Octopus.Environment.Name | Replace " .*" ""}" \
            --query "Stacks[0].Outputs[?OutputKey=='DnsName'].OutputValue" \
            --output text)

        set_octopusvariable "DNSName" $${DNS_NAME}
        echo "DNS Name: $DNS_NAME"

        write_highlight "Open [$${STAGE_URL}index.html]($${STAGE_URL}index.html) to view the frontend web app."
        EOF
        "Octopus.Action.AwsAccount.Variable" = "AWS.Account"
        "Octopus.Action.Script.ScriptSource" = "Inline"
        "Octopus.Action.Aws.AssumeRole" = "False"
        "Octopus.Action.AwsAccount.UseInstanceRole" = "False"
      }
      environments                       = []
      excluded_environments              = []
      channels                           = []
      tenant_tags                        = []
      features                           = []
    }

    properties   = {}
    target_roles = []
  }

Configuring shared variables

Library variable sets provide a way to share variables across many projects. This allows projects to access a common set of variables capturing cross cutting details such as accounts, AWS regions, and names of CloudFormation stacks that multiple projects query for outputs.

To create the AWS account in Octopus, apply the Terraform configuration in the terraform/libraryvariablesets/octopus directory using the following commands: 

cd terraform/libraryvariablesets/octopus
terraform init
terraform apply

You will be prompted for the URL of your Octopus instance, the Octopus API key, and the ID of the space to apply the configuration to. Terraform then displays a list of changes and prompts you to apply them. Once the configuration is applied a new library variable set called Octopub containing three variables is created:

  • AWS.Account - A reference to the account created previously.
  • AWS.CloudFormation.ApiGatewayStack - The name of the CloudFormation stack that will create the shared API Gateway.
  • AWS.Region - The AWS region that the resources are created in.

    • We now have all the global resources required to begin deploying AWS resources. The next step is to deploy a shared API Gateway instance.

Configuring the environments

Octopus environments typically represent copies of infrastructure that deployments are progressed through. For example, you may have a development environment where the engineering teams deploy their applications as they make changes. Once a version of the application is ready to be tested it is promoted to a test environment where QA and product owners can review the new functionality. Once all testing is complete and the business signs off, the application is deployed to the production environment where is made available to end users.

To create the AWS account in Octopus, apply the Terraform configuration in the terraform/environments/octopus directory using the following commands: 

cd terraform/environments/octopus
terraform init
terraform apply

You will be prompted for the URL of your Octopus instance, the Octopus API key, and the ID of the space to apply the configuration to.

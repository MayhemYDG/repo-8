--- --- The Sales Engineering Guide to AWS Lambda Deployments

Prerequisites

To follow along with this book you will need a number of tools installed, as well as having an instance of Octopus available.

The easiest way to get access to an instance of Octopus is to sign up for a cloud instance. You can do so via https://octopus.com/start.

You then need to generate an API key for your Octopus instance, which can be done using the instructions at https://octopus.com/docs/octopus-rest-api/how-to-create-an-api-key.

We'll configure Octopus via Terraform and the Octopus Terraform provider. The documentation at https://developer.hashicorp.com/terraform/downloads provides instructions for installing Terraform on your local operating system.

The Terraform configuration files are exposed via git. The GitHub documentation at https://github.com/git-guides/install-git provides instructions for installing git.

Finally, you will need an AWS subscription to perform the deployments to. Specifically, you will need the Access and Secret keys associated with an AWS account that has permissions to create resources such as API Gateway instances, Lambdas, and S3 buckets.

Once you have the prerequisite tools installed, download a copy of the git repo with the command: 

    git clone https://github.com/mcasperson/AWSLambdaOctopusDeployment.git

This creates a directory called AWSLambdaOctopusDeployment which contains the sample Terraform configuration files referenced throughout the rest of this book.
